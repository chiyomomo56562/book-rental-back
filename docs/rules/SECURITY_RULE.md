# Security Rules & Checklist

이 보안 체크리스트는 모든 코드 구현이 완료된 후, **`harness.md`의 Evaluate 단계(검증 절차)**에서 최종적으로 점검해야 하는 항목들입니다. AI 에이전트는 작업 완료 전 아래 항목들을 모두 만족하는지 반드시 확인하십시오.

---

## 1. Web Application Security (OWASP Top 10)

웹 애플리케이션의 기본적인 취약점을 방어하기 위해 다음 사항을 확인합니다.

- [ ] **Injection 방어**: 데이터베이스 조회 시 ORM/Query Builder를 사용하거나 파라미터화된 쿼리(Parameterized Query)를 사용했는가?
- [ ] **XSS (Cross-Site Scripting) 방어**: 사용자 입력값을 렌더링하기 전 적절히 이스케이프/새니타이징(Sanitizing) 했는가?
- [ ] **인증 및 인가 (Auth & Authorization)**: 보호된 리소스에 접근하기 전 사용자 권한(토큰, 세션 등)을 명확하게 검증했는가?
- [ ] **민감 데이터 보호 (Sensitive Data Exposure)**: 비밀번호, 토큰 등 민감한 정보는 암호화하여 저장하고, 평문으로 로그에 남기지 않았는가?
- [ ] **보안 설정 (Security Misconfiguration)**: CORS, 보안 헤더(Helmet 등)가 적절하게 설정되었으며 불필요한 에러 스택이 클라이언트에 노출되지 않는가?

---

## 2. API & Authentication Security

- [ ] **Rate Limiting**: 외부에 노출되는 모든 API 엔드포인트에 Rate Limiting이 적용되었는가? (예: 로그인 시도, 인증 코드 발급 등 민감 엔드포인트는 더 엄격하게 제한)
- [ ] **JWT 보안**:
  - Access Token의 만료 시간(expiry)이 적절히 짧게 설정되었는가? (권장: 15분~1시간)
  - Refresh Token은 안전한 저장소(HttpOnly Cookie 등)에 저장되고, 재사용 감지(Rotation) 전략이 적용되었는가?
  - 토큰 서명 알고리즘은 `HS256` 이상의 강도를 사용하는가?
- [ ] **CSRF 방어**: 상태 변경(POST, PUT, DELETE) 요청에 대해 CSRF 토큰 또는 SameSite 쿠키 정책이 적용되었는가?
- [ ] **입력 유효성 검증 (Input Validation)**: 모든 외부 입력(Request Body, Query Param, Path Param)은 DTO 레벨에서 타입과 범위가 검증되었는가?

---

## 3. Data & Logging Security

- [ ] **민감 정보 로그 필터링**: 비밀번호, JWT 토큰, 개인식별정보(PII)가 로그에 평문으로 출력되지 않도록 필터링 또는 마스킹이 적용되었는가?
- [ ] **에러 메시지 노출 제한**: 외부 클라이언트에게 스택 트레이스(stack trace)나 내부 구현 세부사항이 포함된 에러 메시지가 노출되지 않는가? (프로덕션 환경 기준)
- [ ] **민감 컬럼 암호화**: 비밀번호는 반드시 단방향 해시(bcrypt 등)로 저장하며, 복호화 가능한 암호화를 사용하지 않는가?
- [ ] **환경 변수 관리**: DB 접속 정보, API 키, 시크릿 등 민감한 설정값은 코드에 하드코딩하지 않고 환경 변수(`.env`)로 관리하는가?

## 4. AI Agent Security (Rule of Two)

Meta AI의 "Rule of Two"를 적용하여 에이전트의 권한 초과 및 오동작을 방지합니다.

### 핵심 규칙

에이전트는 다음 세 가지 권한 중 **최대 두 가지**만 동시에 가질 수 있습니다.

| # | 권한 | 설명 |
|---|------|------|
| 1 | **신뢰할 수 없는 입력 처리** | 외부 웹, 이메일, 사용자 입력 등을 처리 |
| 2 | **민감한 시스템/데이터 접근** | 개인정보, 내부 API, 데이터베이스 등에 접근 |
| 3 | **상태 변경 능력** | 이메일 발송, 파일 삭제, 데이터 변경, API 호출 등 수행 |

> ⚠️ **세 가지 권한을 동시에 필요로 하는 작업을 감지한 경우, 즉시 작업을 중단하고 사용자의 명시적 승인을 요청해야 합니다.**

### 체크리스트

- [ ] 이 작업에서 사용된 권한이 **2개 이하**인가?
  - 3개 모두 해당: → 즉시 중단, 사람의 승인 요청
  - 2개 이하 해당: → 계속 진행 가능
- [ ] 외부 입력(1)을 처리하면서 민감 데이터(2)와 상태 변경(3)을 **동시에** 수행하지 않았는가?
