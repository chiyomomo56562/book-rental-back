# E2E 테스트 명세 (End-to-End)

> **참고 규칙**: `docs/rules/TEST_RULE.md`
> **템플릿**: `docs/tests/test.template.md`

이 문서는 시스템의 핵심 비즈니스 시나리오를 전체적으로 검증하는 E2E 테스트 명세입니다.

---

## 📁 테스트 위치
- `test/e2e/[scenario].e2e.test.ts`

---

## 1. 도서 대여 및 반납 시나리오
**시나리오**: `도서 등록 -> 목록 확인 -> 대여 -> 이력 확인 -> 반납 -> 상태 확인`

### 📋 단계별 검증
1. **도서 등록**: `POST /api/books` 호출 후 성공 응답(201/200) 확인.
2. **도서 대여**: 등록된 ID로 `POST /api/books/:id/rentals` 호출.
3. **이력 확인**: `GET /api/books/:id/rentals`를 통해 대여 정보가 존재하는지 확인.
4. **도서 반납**: `PATCH /api/books/:id/rentals/return` 호출.
5. **최종 확인**: `GET /api/books/:id` 호출 시 상태가 `AVAILABLE`인지 검증.

---

## 🛠 실행 가이드
- **설정**: `vitest.e2e.config.ts`를 사용하여 유닛 테스트와 분리 실행.
- **Teardown**: 각 시나리오 실행 후 생성된 테스트 데이터를 반드시 정리한다.
- **환경**: 실제 애플리케이션 서버를 기동한 상태에서 `supertest`로 외부에서 요청을 보낸다.
